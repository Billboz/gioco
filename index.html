Gioco is a easy to implement gamification gem based on plug and play concept.
Doesn't matter if you already have a full and functional database, Gioco will
smoothly integrate
everything and provide all methods that you might need.
Documentation

Concept
Gioco was built with a Badge, Level and Points concept to reach the majority of gamifications implementation. The Points are optinal but can be very usefull.
The basic logic is, that a Resource ( eg. User ) have Points and when this Resource get determined Points it can win or lose a Level, connecting or disconnecting the Resource with a Badge ( getting a badge through a level ).
But Points are optional, so, if you dont wanna register Points, all you gonna do will be add and remove Badges as your logic requires ( through a level ).

Install and Setup
Gioco is avaible through Rubygems, to install add the gioco in your gemfile:
gem 'gioco'
and run the bundler:
> bundle install

After install gioco you're gonna have to setup it, the MODEL_NAME should be replaced for the model you want to be, as gioco treat, the resource of the gamification. (eg. user )
rails g gioco:setup MODEL_NAME --points;
or
rails g gioco:setup MODEL_NAME;
The optional --points argument will setup the gioco with a points system explained at the Concept

Badges Managment
The badges are managed through Rake tasks, enabling you to add and remove badges as you need:

To add Badges use:
rake gioco:add_badge[BADGE_NAME,POINTS,DEFAULT]
Or if you installed gioco without points option:
rake gioco:add_badge[BADGE_NAME,DEFAULT]
And to remove Badges use:
rake gioco:remove_badge[BADGE_NAME]
The boolean DEFAULT option is responsible to add a specific badge to all your current resources registrations.

Methods
Gioco provides you some methods to use inside of the application that will allow you to easy implement your gamification logic.

Resource
Change Points
Gioco::Resources.change_points( Resource_id, Points, Resource_obj[optional] )
This method only is usefull when you setup the Gioco with the points system. The Resource_obj is an optional argument that can be use to avoid useless queries on database, so instead of pass a Resource_id you can pass it as nil and the resource obeject at the last parameter.

Badges
Add
Gioco::Badge.add( Resource_id, Badge_id, Resource_obj[optional], Badge_obj[optional] )
The Badge.add method is responsable to add a specific badge to some resource, again, the Resource_obj and the Badge_obj are optinal arguments that can be use to avoid useless queries on database, so instead of pass a Resource_id and Badge_id you can pass it as nil and the resource and badge obejects at the lasts parameters.

Remove
Gioco::Badge.remove( Resource_id, Badge_id, Resource_obj[optional], Badge_obj[optional] )
The Badge.remove method is used to remove a badge of a resource, the Resource_obj and the Badge_obj follow the same logic of the others optional arguments of the Add method.

Application
Inside your application you might have to get the Badges and Levels data. And you are able to do this using (replace RESOURCE_NAME for your resource model name) :
RESOURCE_NAME.badges
RESOURCE_NAME.levels
