Gioco is an easy to implement gamification gem based on the concept of "plug and play".
It doesn't matter if you already have a full and functional database, Gioco will
smoothly integrate
everything and provide all methods that you might need.
Documentation

Concept
Gioco implements the concepts of Badge, Level and Points to reach the majority of gamifications implementations. Points are optional but can be very useful.
The basic logic is that a Resource ( eg. User ) has Points and when this Resource get a certain amount of Points it can win or lose Levels, connecting or disconnecting the Resource with a Badge ( getting a badge through a level ).
But Points are optional so, if you don't want to register Points, all you have to do is to add and remove Badges as required by your business rules ( through a level ).

Install and Setup
Gioco is available through Rubygems, to install add Gioco in your gemfile:
gem 'gioco'
and run Bundler:
> bundle install

After installing Gioco you'll need to set it up. The MODEL_NAME should be replaced for the model you want the resource of the gamification to be. (eg. user )
rails g gioco:setup MODEL_NAME --points;
or
rails g gioco:setup MODEL_NAME;
The optional --points argument will setup the gioco with a points system explained at the Concept

Badges Management
The badges are managed through Rake tasks, enabling you to add and remove badges as you need:

To add Badges, use:
rake gioco:add_badge[BADGE_NAME,POINTS,DEFAULT]
Or, if you installed gioco without points option:
rake gioco:add_badge[BADGE_NAME,DEFAULT]
And to remove Badges, use:
rake gioco:remove_badge[BADGE_NAME]
The boolean DEFAULT option is responsible to add a specific badge to all your current resource registrations.

Methods
Gioco provides you with some methods to use inside of the application that will allow you to easily implement your gamification logic.

Resource
Change Points
Gioco::Resources.change_points( Resource_id, Points, Resource_obj[optional] )
This method is useful only when you setup the Gioco with the points system. The Resource_obj is an optional argument that can be used to avoid useless queries on database. So, instead of passing a Resource_id you can pass it as 'nil' and the resource object at the last parameter.

Badges
Add
Gioco::Badge.add( Resource_id, Badge_id, Resource_obj[optional], Badge_obj[optional] )
The Badge.add method is responsible to add a specific badge to a resource, the Resource_obj and the Badge_obj are optional arguments that can be used to avoid useless queries on the database. So, instead of passing a Resource_id and Badge_id you can pass it as 'nil' and the resource and badge objects at the last parameters.

Remove
Gioco::Badge.remove( Resource_id, Badge_id, Resource_obj[optional], Badge_obj[optional] )
The Badge.remove method is used to remove a badge of a resource, the Resource_obj and the Badge_obj follow the same logic of the others optional arguments of the Add method.

Application
Inside your application you might have to get Badges and Levels data. And you are able to do this using (replace RESOURCE_NAME for your resource model name) :
RESOURCE_NAME.badges
RESOURCE_NAME.levels
